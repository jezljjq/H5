### 识别链路一致性

- 测试识别和正式运行统一走 `h5bot.recognition.recognize_step()`。
- 统一使用右侧“全局识别参数”中的默认识别阈值和默认重试次数。
- 保留 `FlowStep.threshold` / `FlowStep.retries` 兼容旧配置，但当前 UI 不显示、不编辑，当前识别不使用它们覆盖全局参数。
- 识别日志统一输出操作来源、hwnd、窗口标题、步骤名称、模板列表、ROI、阈值、后端、命中状态、命中模板、命中坐标和失败原因。

### ROI 可视化

- 右侧当前步骤详情支持选择 ROI、自动生成 ROI、清空 ROI。
- ROI 使用窗口客户区坐标 `x1,y1,x2,y2`。
- 自动生成 ROI 失败时只提示原因，不修改原 ROI。
- ROI 保存后同步流程表、右侧详情、配置对象和 `config/app_config.json`。

### 运行前配置检查

- 新增 `h5bot/preflight.py`。
- 开始运行前检查窗口、任务、启用步骤、ROI、跳转目标、模板文件和大漠可用性。
- 模板缺失只提示，不删除引用，不自动修改配置。

### 自动抢拍任务框架

- 自动抢拍需求已正式下发；本阶段只实现框架，不做多窗口并发、OCR 或真实抢拍压测。
- 新增 `h5bot/auction_config.py`，定义 `AuctionTaskConfig` 和任务类型常量。
- 新增 `h5bot/auction.py`，提供单窗口 `AuctionRunner` 状态机框架和按钮 ROI 计算。
- `TaskBranch` 新增 `task_type` 和 `auction_config`，普通流程任务默认仍为 `flow`。
- 普通流程任务继续由 `FlowRunner` 执行，自动抢拍任务独立使用 `AuctionRunner`。
- UI 增加任务类型入口和自动抢拍配置面板：
  - 进入拍卖界面。
  - 目标物品模板组。
  - 灰色一口价按钮模板组。
  - 可点击一口价按钮模板组。
  - 确认按钮模板组。
  - 拍卖列表识别区域和确认按钮识别区域。
  - 按钮偏移、频率滚动、运行策略。
  - 测试自动抢拍配置、单窗口启动、停止当前任务。
- 新增 `tests/test_auction.py`，覆盖自动抢拍配置、序列化、按钮 ROI、状态机关键分支和普通流程兼容。

### 自动抢拍 UI 工作台优化

- 自动抢拍配置新增拍卖入口图标模板组、拍卖界面确认模板组、拍卖入口识别区域、拍卖界面确认区域。
- 自动抢拍状态机调整为 S0-S7，新增 S1“进入拍卖界面”：
  - 先检查是否已在拍卖界面。
  - 已在拍卖界面时跳过入口点击。
  - 未在拍卖界面时识别并点击拍卖入口，再通过拍卖界面确认模板组验证页面进入成功。
- UI 按任务类型拆分工作台：
  - 普通流程任务显示流程路径、流程步骤表、步骤操作按钮和当前步骤详情。
  - 自动抢拍任务在中间主工作区显示自动抢拍配置主面板，右侧显示全局识别参数、自动抢拍运行状态和运行日志。
- 自动抢拍配置从右侧移到中间主工作区，减少右侧负担和主工作区空白。
- 用户可见文案已中文化，不在界面显示英文配置字段名。

### 窗口任务队列与混合调度

- 新增 `h5bot/window_tasks.py`，统一管理窗口任务队列项、任务类型显示名和按队列项创建执行器。
- 新增 `window_task_queues`，运行模型调整为 `窗口 -> 任务队列 -> 队列任务`。
- 队列项保存方案名、任务名、任务类型、启用状态、顺序和第一版预留执行策略字段。
- 旧 `window_task_bindings` 会兼容迁移为该窗口任务队列中的第一个任务，不删除旧方案、旧任务和模板引用。
- 方案不再作为窗口运行时强制层级，只作为任务模板分类、预设集合和快速添加到窗口队列的来源。
- 中间主区域增加当前窗口任务队列，支持添加、批量添加、替换选中窗口队列、移除、上移、下移、启用/禁用、清空和批量清空。
- 中间上半区按新 UI 原型重排为左侧“可选任务（模板库）”、中间队列操作按钮、右侧“当前窗口任务队列”。
- 主界面移除方案下拉框、新增方案、重命名方案、删除方案等方案层级入口；方案只作为内部兼容数据和任务模板来源保留。
- 任务模板库可见项只显示任务名和任务类型，不再显示方案来源。
- 新增“复制当前窗口队列到选中窗口”，用于把一个窗口的完整队列复制给多个窗口。
- 当前窗口任务队列中选中的任务会驱动下方配置区，普通流程任务显示普通流程配置，自动抢拍任务显示自动抢拍配置。
- 选中自动抢拍任务时，中间主工作区隐藏普通流程路径、普通流程步骤表、模板目录提示和普通流程当前步骤详情。
- 右侧辅助区按任务类型切换：普通流程任务显示当前步骤详情；自动抢拍任务显示自动抢拍运行状态和配置完整性提示。
- “开始全部”改为遍历窗口自己的启用队列，同一窗口内按顺序执行，不同窗口之间仍按现有线程式调度并行。
- 队列任务按任务类型选择执行器：普通流程任务使用 `FlowRunner`，自动抢拍任务使用 `AuctionRunner`。
- 左侧窗口列表显示窗口标题、队列数量、当前执行任务和窗口状态。
- 同一窗口仍保留运行互斥保护，重复启动同一窗口时提示“该窗口已有任务正在运行，请先停止当前任务。”
- 普通流程和自动抢拍日志统一带窗口标题和任务类型前缀，例如 `[斗罗大陆H5-1][普通流程任务]`、`[斗罗大陆H5-2][自动抢拍任务]`。
